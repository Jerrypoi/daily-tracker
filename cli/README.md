# daily-tracker-cli

Command-line client for the [Daily Tracker](../README.md) backend. Designed to be invoked via `npx` from scripts and LLM agents.

- TypeScript source, zero runtime dependencies (Node 18+ for built-in `fetch`).
- Every successful command emits a single JSON document to stdout.
- Every failure emits a single JSON document to stderr and exits non-zero.

## Layout

The npm package lives at the **repo root** (so npx can install it directly from the GitHub URL without needing a subdirectory specifier):

```
daily-tracker/         repo root — also the npm package root
├── package.json       declares "daily-tracker" bin + prepare script
├── tsconfig.json      rootDir cli/src, outDir cli/dist
├── cli/
│   ├── src/                 TypeScript sources
│   │   ├── bin.ts                # entrypoint (#!/usr/bin/env node)
│   │   ├── index.ts              # command dispatch
│   │   ├── args.ts               # arg parser
│   │   ├── client.ts             # fetch wrapper + ApiError
│   │   └── help.ts
│   ├── dist/                generated by `tsc` (gitignored)
│   └── README.md            this file
└── skills/                  Claude Code agent skills (sibling of cli/)
```

## Install / invoke

Run directly from the GitHub source — no clone required. `npx` installs the repo, the `prepare` script compiles `cli/src/` → `cli/dist/`, then the bin runs:

```bash
npx --yes github:Jerrypoi/daily-tracker daily-tracker help
```

Pin a tag or commit for reproducibility:

```bash
npx --yes github:Jerrypoi/daily-tracker#v0.1.0 daily-tracker help
npx --yes github:Jerrypoi/daily-tracker#<sha>  daily-tracker help
```

Or install globally once:

```bash
npm install -g github:Jerrypoi/daily-tracker
daily-tracker help
```

Local dev (only when working on the CLI itself):

```bash
npm install         # builds via `prepare`
npm run build       # rebuild
npm run typecheck   # tsc --noEmit
node cli/dist/bin.js help
```

## Configuration

| Variable | Purpose | Default |
|---|---|---|
| `DAILY_TRACKER_API_KEY` | API key (Bearer). Recommended for agents. | — |
| `DAILY_TRACKER_JWT` | JWT (only for `/api-keys` endpoints). | — |
| `DAILY_TRACKER_API_URL` | API base URL. | `http://localhost:8080/api/v1` |

To mint an API key see `../skills/daily-tracker-setup/SKILL.md`.

## Quick reference

```bash
daily-tracker whoami

daily-tracker topics list
daily-tracker topics create --name "deep work" --color "#3b82f6"
daily-tracker topics update 12 --name "focus" --color "#ef4444"

daily-tracker tracks list --start 2026-04-20 --end 2026-04-27
daily-tracker tracks create --start-time 2026-04-27T09:00:00Z --topic 12 \
  --comment "review PRs"
daily-tracker tracks delete 87
```

Run `daily-tracker help` for the full command list.

## Use from an LLM agent (Claude Code skills)

Two skills are bundled at the repo root under `skills/`:

- `daily-tracker/SKILL.md` — day-to-day topic + track operations.
- `daily-tracker-setup/SKILL.md` — one-time register / login / mint-API-key flow.

Install them by symlinking into your Claude Code skills directory (run from the repo root):

```bash
ln -s "$(pwd)/skills/daily-tracker"        ~/.claude/skills/daily-tracker
ln -s "$(pwd)/skills/daily-tracker-setup"  ~/.claude/skills/daily-tracker-setup
```

Or copy them into a project's `.claude/skills/` directory if you want them scoped to one repo.

## Output contract

Success — single JSON document on stdout:

```json
{
  "id": 12,
  "topic_name": "deep work",
  "display_color": "#3b82f6",
  "created_at": "2026-04-27T09:00:00Z",
  "updated_at": "2026-04-27T09:00:00Z"
}
```

Failure — single JSON document on stderr, non-zero exit:

```json
{
  "error": "VALIDATION_ERROR",
  "status": 400,
  "message": "start_time must be at :00 or :30 minutes",
  "body": { "error": "VALIDATION_ERROR", "message": "..." }
}
```

For HTTP 204 (delete/revoke) the CLI emits `{"deleted": <id>}` or `{"revoked": <id>}` so downstream tools always parse JSON.
